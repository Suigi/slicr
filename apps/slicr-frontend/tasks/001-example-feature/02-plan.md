# Example Feature Plan

## Summary

This is a minimal example showing the expected shape of a feature plan file created during Phase 2.

## Goals

- Add one small, user-visible behavior.
- Keep the example short enough to copy.

## Non-Goals

- No real implementation details.
- No unrelated cleanup.

## Core Rule

- The feature should describe one stable behavioral rule in plain language.

## Data Model

- Describe any new derived data or contract additions here.

## Implementation Shape

- Name the layer that will change first.
- Explain the order of later changes.

## Test Plan

- Identify the focused regression that should be added first.
